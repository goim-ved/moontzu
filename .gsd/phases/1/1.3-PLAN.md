# Plan 1.3: Database RLS Schema Setup

## Objective
Establish the foundational multi-tenant PostgreSQL schema within Supabase (or locally for Supabase), including the `tenants`, `profiles`, and `tenant_members` tables, and apply rigorous Row-Level Security (RLS) policies.

## Context
- .gsd/SPEC.md

## Tasks

<task type="auto">
  <name>Initialize Database Schema Definitions</name>
  <files>supabase/migrations/0000_core_setup.sql</files>
  <action>
    - Create a SQL migration file at `supabase/migrations/0000_core_setup.sql`. 
    - Define a `tenants` table (id, name, slug/subdomain, created_at).
    - Define a `profiles` table (id references auth.users, name).
    - Define a `tenant_members` table (id, tenant_id, user_id, role [owner, admin, staff], created_at).

  <name>Implement RLS Policies</name>
  <files>supabase/migrations/0000_core_setup.sql</files>
  <action>
    - Ensure `ALTER TABLE x ENABLE ROW LEVEL SECURITY;` is present for all tables.
    - Write RLS policies for `tenants` (Users can view their own tenant).
