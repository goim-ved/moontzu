# Plan 1.2: Supabase Authentication Setup

## Objective
Configure Supabase for Server-Side Rendering (SSR) Authentication, defining the necessary server, client, and middleware clients, and creating the basic login/register pages.

## Context
- .gsd/SPEC.md
- src/app/layout.tsx

## Tasks

<task type="auto">
  <name>Install Supabase Dependencies</name>
  <files>package.json</files>
  <action>
    - Run `npm install @supabase/supabase-js @supabase/ssr`
    - Setup generic placeholder `.env.local` file with standard Supabase ENV vars (`NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY`).
  </action>
  <verify>Get-Content package.json | Select-String "supabase/ssr"</verify>
  <done>Supabase SSR packages installed.</done>
</task>

<task type="auto">
  <name>Create Supabase SSR Clients & Middleware</name>
  <files>src/utils/supabase/server.ts, src/utils/supabase/client.ts, src/utils/supabase/middleware.ts, src/middleware.ts</files>
  <action>
    - Create `src/utils/supabase/server.ts` function that creates a ServerClient according to Supabase SSR documentation (using next/headers `cookies`).
    - Create `src/utils/supabase/client.ts` function that creates a BrowserClient.
    - Create `src/utils/supabase/middleware.ts` to refresh tokens using Supabase SSR `createServerClient`.
    - Export the middleware function from `src/middleware.ts` to apply it to Next.js routes (excluding statics).
  </action>
  <verify>npm run build</verify>
  <done>Supabase helper functions and Next.js middleware compile successfully.</done>
</task>

<task type="auto">
  <name>Create Auth Routes</name>
  <files>src/app/login/page.tsx, src/app/auth/callback/route.ts</files>
  <action>
    - Implement a basic unified Login/Signup UI at `src/app/login/page.tsx` using Shadcn components (email/password). Include action functions that log into Supabase.
    - Create the Auth Callback route at `src/app/auth/callback/route.ts` which handles the code exchange and redirects back to the main app dashboard.
  </action>
  <verify>npm run build</verify>
  <done>Authentication UI forms and the callback route compile and pass linting.</done>
</task>

## Success Criteria
- [ ] Supabase SSR configuration is complete.
- [ ] Next.js middleware is actively intercepting requests.
- [ ] Login and Registration UI forms are built.
