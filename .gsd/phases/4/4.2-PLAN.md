# Plan 4.2: Stripe Connect Onboarding

## Objective
Build the functionality for a Tenant Owner to integrate their platform presence with Stripe using Stripe Connect standard/express onboarding APIs. We'll set up a settings page to initialize this.

## Context
- .gsd/SPEC.md
- src/app/[domain]/(dashboard)/settings

## Tasks

<task type="auto">
  <name>Dashboard Settings Page</name>
  <files>src/app/[domain]/(dashboard)/settings/page.tsx</files>
  <action>
    - Create a bare-bones settings page `src/app/[domain]/(dashboard)/settings/page.tsx`.
    - Retrieve the tenant. Check if `stripe_account_id` currently exists.
    - If no Stripe Account ID exists, show a button to run the onboarding action.
    - Update the Dashboard Layout sidebar to include the Settings link.
  </action>
  <verify>npm run build</verify>
  <done>Settings UI correctly identifies Stripe onboarding status and gives access to the connect action.</done>
</task>

<task type="auto">
  <name>Stripe Onboarding Server Action</name>
  <files>src/app/[domain]/(dashboard)/settings/actions.ts</files>
  <action>
    - Ensure `STRIPE_SECRET_KEY` usage placeholder exists (you can mock the Stripe calls if the local ENV key is not set, or wrap it in a try-catch for demo stability if keys are missing from `.env.local` during execution).
    - Create action `connectStripe`.
    - Flow:
        1. Query `stripe_account_id` from `tenants`.
        2. If empty, create an account using `stripe.accounts.create({ type: 'standard' })`.
        3. Save the new `stripe_account_id` back to the `tenants` table.
        4. Generate `stripe.accountLinks.create` to get an onboarding URL.
        5. Redirect user to that generated Stripe URL.
  </action>
  <verify>Get-Content "src/app/[domain]/(dashboard)/settings/actions.ts" | Select-String "stripe.accountLinks.create"</verify>
  <done>Action properly interfaces with Stripe API and Supabase syncing the Account ID.</done>
</task>

## Success Criteria
- [ ] Tenant owners can instantiate a Stripe Connect relationship seamlessly securely recording their ID.
